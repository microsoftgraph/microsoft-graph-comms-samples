    Setting up Stateful Bots with the Graph Calling SDK

    This document explains how to setup the Graph Calling SDK for using in other stateful projects. This documentation closely follows the samples available along with it.

    The Graph Calling SDK is exposed as the IStatefulClient interface. Any bot created is an instance of the IStatefulClient interface.

    Stateful Calling Bots

    This Graph Calling SDK supports calling abilities to bots in stateful scenarios. They are called Local Media bots.

    As the name suggests, Local Media bots media is hosted by the bot itself, i.e., the bot sends and receives raw Audio and Video frames as required.

    Important

    Local Media calls require that the further notifications and updates from the server always hit the same instance of the bot when multiple instances are present. Look at State Management to learn more

    Create a New Bot

    Note

    Take a look at Register calling bots.

    • Register your bot in the Bot framework portal and save the Microsoft AppId and Password as entered in it.
    • Add the Skype channel, Teams channel and the settings required for that. You would select your media type to be Local Media - Real Time Media in the settings. You also provide a webhook where Skype would contact you for the incoming notifications.
    • After the setup is done, the bot should create an IStatefulClientBuilder object on which you can set all the required parameters like the authentication provider (SetAuthenticationProvider), notification url (SetNotificationUrl) and others.
    • Call StatefulClientBuilder.Build method to create a IStatefulClient instance which represents the Graph Stateful SDK.
